## Main Readability Strains

### 1. The README Is Accurate But Heavy

The README now contains the right elements, but it is doing too many jobs at once:

- first-entry orientation;
- whole-archive index;
- claim boundary note;
- public/private boundary note;
- sequence map;
- reference map;
- presentation and companion archive routing.

This is not wrong. It is just a lot of load for a first screen.

Better flow would separate:

- **Start Here** - 3 to 5 entry points for a new reader;
- **Reader Routes** - new reader, practitioner, researcher, intimate/private boundary, archive curator;
- **Reference Layers** - Dictionary, taxonomy, translation map;
- **Full Index** - the complete list, lower on the page or moved to a dedicated `INDEX.md`.

The current full index should not disappear. It should stop being the first thing the reader has to metabolize.

### 2. `where-to-begin.md` Needs To Catch Up

`00-orientation/where-to-begin.md` still reads from the earlier architecture. It says the history comes first, the myth comes next, and the empirical record comes last. That was true for the previous reading path.

Now the archive begins with:

- the archive learning to read itself;
- biological memory;
- claim environments;
- protective public/private boundary.

`where-to-begin.md` should be revised as the true gateway. It should explain:

- why the archive begins with claim architecture now;
- how to use the Dictionary without turning the work into bureaucracy;
- how to read public material while knowing that a private layer exists;
- which path to follow depending on whether the reader is new, technical, symbolic, or already in practice.

This one file can do a lot of flow repair without moving many other files.

### 3. Orientation Is Too Long To Be One Continuous Road

The Orientation section has 29 files. It is powerful, but the current presentation makes it feel like a single required pilgrimage.

It wants internal grouping.

Possible groups:

- **Suppression and Lineage:** Women of Alchemy, Beauty and the Binding, In the Name of Love, Why the Distillation Is Missing, Grief, Language of Erasure.

This can be done first in the README without renaming files. Folder moves can wait.

### 4. The Dictionary Is Clear But Not Yet Fully Usable

The Dictionary is strong as a conceptual layer. The pages are short, restrained, and protective.

What it needs next is use-in-context.

Each term page would become easier to use if it included:

- **What this word does not claim**
- **Related archive pages**
- **Example sentence: measured register**

This would make the Dictionary less like a static glossary and more like a reader's handrail.

### 4. Add Dictionary Use Examples

Do not make the Dictionary bigger by adding abstractions.

Make it more usable by showing how terms behave in different registers.

Example:

- "Solar measured as pH/ORP/EC under these conditions" belongs to Instrument Table.
- "Solar as fire, ascent, and volatility" belongs to Comparative Archive or Mythic Field.
- Neither sentence needs to impersonate the other.

### 5. Add Claim-Posture Blocks To Mixed Documents

Use the new architecture on the pages where it matters most.

Start with synthesis and analysis, not with every file.

### 6. Delay Folder Reorganization

The urge to reorganize folders is understandable, but the archive does not need that yet.

First improve the maps.

If the maps solve the reader's movement, file moves may be unnecessary.

## What To Protect

Do not let the machine flatten the living voice into a database.

The archive needs structure, but the structure is only a reading ethic. It is not the authority of the work.

Protect:

- the single-subject frame;
- the poetic and symbolic register;
- the body's lived intelligence;
- uncertainty;
- private intimate records;
- the difference between observation and instruction;
- the difference between comparison and proof.

The goal is not to make the archive sound institutional.

The goal is to make it harder to misunderstand.
